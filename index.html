Dynamic-Object-Persistence by kaniska

Store incoming objects dynamically in Database

1. Same type of Objects with fixed set of attributes should be dynamically stored using SimpleJdbcInsert command. In this approach - - (i) no need to maintain - hibernate mapping files , (ii) no need to create pojo / xsd , (ii) as a result no conversion layer - wsdl / http stream to pojo 2. Same type of Objects with dynamically generated attributes can be effortlessly stored using MongoDB.
